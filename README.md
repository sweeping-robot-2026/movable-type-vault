# 活字库（Movable Type Vault）

**你的私人数字人调度核心。**

现在的数字AI太死板了。它们统一、礼貌、安全，但没有性格，没有记忆，不属于你。

活字库想做的，是一套完全不同的AI架构：**用一堆独立的、可插拔的“活字模块”，拼出一个只属于你的、能自我进化的数字人。**

- **模块化**：逻辑脑、风格脑、情感脑是独立的，可以单独升级、替换、定制。
- **本地优先**：全部跑在你的机器上，数据不离开你的硬盘。
- **越用越像你**：通过日常使用形成你的个人风格，也可以喂数据变成夸张的本地知识库。
- **自带免疫**：一个叫“虚拟淋巴结”的独立进程，全程旁听，帮你挡掉恶意诱导。

> 这不是一个通用大模型。这是你的数字人的调度核心。

## 架构

```
用户输入
   │
   ▼
[路由层] ──────────────────────┐
   │                            │
   ├─→ 活字A (逻辑) ── 监视点 ──┤
   ├─→ 活字B (风格) ── 监视点 ──┤
   ├─→ 活字C (情感) ── 监视点 ──┤
   │                            │
   ▼                            │
[组装层] ──────── 监视点 ──────┤
   │                            │
   ▼                            ▼
输出 ←──────────────── [虚拟淋巴结]
```

- **活字**：独立的推理模块，每个只做一件事。
- **组装层**：把各个活字的输出编织成一段完整的回复。
- **虚拟淋巴结**：不参与生成，只负责全程旁听和安全介入。

## 快速开始

```bash
# 克隆仓库
git clone https://github.com/sweeping-robot-2026/movable-type-vault.git
cd movable-type-vault

# 安装依赖
pip install -r requirements.txt

# 下载默认活字模块
python scripts/download_default_types.py

# 启动活字库
python vault.py --config configs/default.yaml
```

默认配置会加载三个基础活字：通用推理、基础风格、情绪感知。你可以随时替换或新增自己的活字。

## 自定义你的数字人

活字库本身不绑定任何特定模型。你可以：

1. **替换活字**：用自己微调的模型替换 `types/` 下的任何模块。
2. **新增活字**：按照接口规范添加新活字，路由层会自动识别。
3. **训练风格**：用你的聊天记录、日记、邮件微调一个风格活字，放进 `types/custom/`。
4. **构建知识库**：把你的文档喂进本地向量数据库，配置文件中启用即可。

## 体量说明

| 组件 | 典型大小 |
|------|---------|
| 调度核心 (本项目) | < 50MB |
| 默认活字模块 (量化) | 4-8GB |
| 风格插件 (LoRA) | 几十到几百MB |
| 本地知识库 | 取决于你喂多少 |

建议运行环境：16GB以上内存，支持NPU/GPU加速更佳。
